## 🎯 设计目标

将当前的功能性页面改造成**真正的游戏界面**，让用户一眼就能感受到：
- 这是一个德州扑克游戏
- 有沉浸感和代入感

## 📋 必须实现的改进（按优先级）

### 1. 牌桌页面（最高优先级）

#### 当前问题：
- ❌ 只是一个白色背景的表单页面
- ❌ 没有牌桌的感觉
- ❌ **玩家位置是随意画在界面上的，不是围坐在牌桌旁**
- ❌ 没有任何游戏氛围

#### 必须改进：

**🎯 核心要求：玩家必须围坐在真实的牌桌旁边**

1. **真实的牌桌**
   - 绘制一个**真实的椭圆形牌桌**（不是方形，不是圆形）
   - 牌桌要有明确的边界和轮廓
   - 深色背景（藏蓝色 `#1A2C3D` 或传统绿色）
   - 丝绒/毛毡材质纹理
   - 边缘暗角效果（营造深度感）
   - 柔和的顶灯光照效果

2. **玩家座位布局（关键！）**
   - **玩家必须围绕牌桌边缘排列**
   - 不是随意放在界面上
   - 椭圆形牌桌的座位分布：
     ```
            座位2    座位3
       座位1              座位4
       
       座位0              座位5
            (玩家)
     ```
   - 每个座位的位置要精确计算：
     - 使用椭圆方程计算座位坐标
     - 座位围绕牌桌中心均匀分布
     - 座位朝向牌桌中心
   - 座位要紧贴牌桌边缘（不是远离牌桌）

3. **座位视觉设计**
   - 每个座位是一个独立的区域
   - 包含：
     - 玩家头像（圆形）
     - 昵称
     - 筹码量（带筹码图标）
     - 当前下注（筹码堆在座位前，靠近牌桌）
     - Dealer/SB/BB 标记（按钮图标）
   - 座位背景：半透明卡片
   - 座位朝向牌桌中心

4. **牌桌中央区域**
   - 公共牌区域（5张牌横向排列）
   - 底池显示（筹码堆在中央）
   - 当前阶段显示（Preflop/Flop/Turn/River）
   - 所有元素都在牌桌内部

5. **下注筹码的位置**
   - 玩家下注的筹码要放在**座位和牌桌中央之间**
   - 不是放在座位上
   - 筹码要"飞向"牌桌中央
   - 形成一个从座位到中央的筹码路径

6. **空间层次**
   - 背景层：房间/环境
   - 牌桌层：椭圆形牌桌
   - 座位层：玩家围坐
   - 筹码层：下注筹码
   - 卡牌层：公共牌和手牌
   - UI 层：按钮和控制

**参考示意图（文字描述）：**
```
┌─────────────────────────────────────────┐
│         房间背景（深色）                  │
│                                         │
│      ╭─────────────────────╮            │
│     ╱   座2      座3        ╲           │
│    │  座1              座4   │          │
│    │                         │          │
│    │        [公共牌]          │          │
│    │        [底池]            │          │
│    │                         │          │
│    │  座0              座5   │          │
│     ╲  (玩家)                ╱           │
│      ╰─────────────────────╯            │
│                                         │
│    [行动按钮区域]                        │
└─────────────────────────────────────────┘
```

**技术实现建议：**
- 使用 CSS `position: absolute` 精确定位座位
- 使用三角函数计算椭圆上的座位坐标
- 使用 CSS `transform: rotate()` 让座位朝向中心
- 使用 SVG 或 Canvas 绘制牌桌轮廓

3. **扑克牌视觉**
   - 真实的扑克牌样式（不是文字）
   - 卡牌阴影和立体感
   - 背面设计（对手的牌）
   - 正面清晰可见（自己的牌）

4. **筹码视觉**
   - 真实的筹码图标（不是数字）
   - 不同面额不同颜色
   - 堆叠效果
   - 筹码飞向底池的动画

5. **玩家信息展示**
   - 头像（圆形）
   - 昵称
   - 筹码量（带筹码图标）
   - 当前下注（筹码堆）
   - Dealer/SB/BB 标记（按钮图标）

6. **当前行动者高亮**
   - 座位发光效果
   - 倒计时圈（环形进度条）
   - 脉冲动画

7. **行动按钮**
   - 大号、醒目的按钮
   - 不同行动不同颜色
     - Fold: 灰色/红色
     - Check: 蓝色
     - Call: 绿色
     - Raise: 橙色/金色
     - All-in: 红色/金色
   - 悬停效果
   - 点击反馈

8. **下注滑块**
   - 视觉化的滑块（不是默认样式）
   - 预设按钮（Min/1/2 Pot/3/4 Pot/Pot）
   - 实时显示下注金额

## 📦 技术要求

### 1. 响应式设计（必须）⚠️

**重要：游戏界面必须同时兼容手机和 PC 端**

#### PC 端（桌面）
- 分辨率：1920x1080 及以上
- 布局：横向椭圆形牌桌
- 玩家座位：6-9 人围坐
- 操作：鼠标点击 + 键盘快捷键

#### 移动端（手机）
- 分辨率：375x667 (iPhone SE) 到 428x926 (iPhone 14 Pro Max)
- 布局：竖屏优化
  - 牌桌缩小，适应屏幕
  - 玩家座位紧凑排列
  - 按钮放大，易于触控
- 操作：触摸友好
  - 按钮最小尺寸：44x44px
  - 滑块易于拖动
  - 避免悬停效果（改用点击）

#### 平板端（可选）
- 分辨率：768x1024 (iPad) 及以上
- 布局：介于 PC 和手机之间
- 支持横屏和竖屏

### 2. 布局适配策略

#### 断点设计
```css
/* 手机 */
@media (max-width: 767px) {
  /* 竖屏布局 */
  /* 按钮放大 */
  /* 字体调整 */
}

/* 平板 */
@media (min-width: 768px) and (max-width: 1023px) {
  /* 中等布局 */
}

/* PC */
@media (min-width: 1024px) {
  /* 完整布局 */
}
```

#### 关键适配点

**牌桌布局：**
- PC：椭圆形，玩家围坐
- 手机：紧凑型，玩家上下排列
- 自动调整座位间距

**扑克牌：**
- PC：标准尺寸（70x100px）
- 手机：缩小（50x70px）
- 保持清晰可读

**筹码：**
- PC：真实堆叠效果
- 手机：简化显示，数字为主

**按钮：**
- PC：标准尺寸（120x40px）
- 手机：放大（150x50px），易于触控
- 间距增加，避免误触

**下注滑块：**
- PC：精细控制
- 手机：大号滑块，易于拖动
- 预设按钮更大

**文字：**
- PC：14-16px
- 手机：16-18px（更大，易读）

### 3. 触控优化（移动端）

- **触控目标：** 最小 44x44px
- **手势支持：**
  - 点击：选择、确认
  - 长按：查看详情（可选）
  - 滑动：调整下注金额
- **避免悬停：** 所有悬停效果改为点击
- **反馈明确：** 触控时有视觉反馈（高亮、震动）

### 4. 性能优化

- **动画流畅：** 60fps（PC 和手机）
- **资源优化：**
  - 图片懒加载
  - 移动端使用小尺寸图片
  - CSS 动画优先（避免 JS 动画）
- **网络优化：**
  - WebSocket 断线重连
  - 移动网络适配（3G/4G/5G）

### 5. 浏览器兼容

- **PC：**
  - Chrome（主要）
  - Firefox、Safari、Edge（次要）
- **移动：**
  - iOS Safari（主要）
  - Android Chrome（主要）
  - 微信内置浏览器（考虑）

### 6. 测试要求

- **设备测试：**
  - iPhone（多种尺寸）
  - Android 手机（多种尺寸）
  - iPad
  - PC（多种分辨率）
- **方向测试：**
  - 竖屏（主要）
  - 横屏（次要）
- **网络测试：**
  - WiFi
  - 4G/5G
  - 弱网环境
