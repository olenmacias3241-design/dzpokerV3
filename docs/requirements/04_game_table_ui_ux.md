# 04 - 牌桌 UI/UX 规格

## 5.1 界面元素
- **玩家座位**: 必须清晰展示玩家头像、昵称、剩余筹码量、当前下的注。
- **当前行动者**: 座位必须有高亮效果（如发光、倒计时圈），并且倒计时要清晰可见。
- **行动按钮**:
  - `Fold`, `Check`, `Call`, `Bet`, `Raise`, `All-in`。
  - **动态显示**:
    - 前无人下注: 显示 `Check`, `Bet`。
    - 前有人下注: 显示 `Fold`, `Call`, `Raise`。
    - `Call` 按钮上应显示需要跟注的具体金额。
    - `All-in` 按钮始终可见，除非玩家筹码为0。
- **下注滑块 (Raise Slider)**:
  - 最小值: 最小加注额 (通常是前一个加注额的两倍)。
  - 最大值: 玩家的总筹码 (All-in)。
  - **预设按钮**: `Min Raise`, `1/2 Pot`, `3/4 Pot`, `Pot Size`。点击后自动移动滑块。
- **底池**:
  - 主底池和所有边池的金额应分开清晰显示。
- **公共牌**: 牌面清晰，发出时有动画效果。
- **玩家底牌**: 仅对自己可见，可随时点击查看。牌局结束后，所有参与比牌的玩家底牌都会翻开。
- **牌面（卡牌）背景**：牌面（含公共牌与玩家底牌）的**背景须为花纹/纹理**，不得使用纯色背景；花纹风格可与主题或卡背统一，保证可读性与质感。
- **赢家提示**:
  - 牌局结束后，系统应高亮获胜玩家的座位。
  - 同时，在公共牌和获胜者底牌中，高亮显示组成最大牌型的5张牌。
  - **牌型名称（如「葫芦」「Full House」）仅显示在当前玩家自己的控制区域**（如自己座位旁或底部信息栏），不对其他玩家展示；其他玩家仅能看到翻开的牌与高亮，不显示牌型文字。

## 5.1.1 牌桌布局与聊天

- **聊天区域**：聊天面板（消息列表与输入框）**固定放置在牌桌页面的右侧**；可折叠/展开，折叠后不占用主桌区域。布局上牌桌主体在左或居中，聊天在右，适配不同分辨率时保持「聊天在右」的相对关系。

## 5.2 交互与动画
- **发牌动画**: 卡牌从牌堆飞向每个玩家。
- **下注动画**: 筹码从玩家座位飞向底池区域。
- **弃牌动画**: 玩家的底牌变灰并翻面。
- **分配底池**: 筹码从底池飞向获胜玩家。
- **声音效果**: 配备下注、弃牌、发牌、胜利等关键操作的音效。
