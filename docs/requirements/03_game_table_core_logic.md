# 03 - 游戏核心逻辑

**完整玩法与打牌规则**见 **[docs/knowledge/TEXAS_HOLDEM_PLAY_AND_RULES.md](../knowledge/TEXAS_HOLDEM_PLAY_AND_RULES.md)**（标准德州扑克玩法与打牌需求文档）。下文为流程摘要与状态机。

## 4.0 一局流程（标准德州扑克）

每一局的顺序固定为：

1. **发 2 张手牌**（底牌）→ **玩家下注一轮**（底牌圈 Preflop）。
2. **发 3 张公共牌**（翻牌 Flop）→ **大家再下一轮注**（翻牌圈）。
3. **发 1 张转牌**（Turn）→ **再下一轮注**（转牌圈）。
4. **发 1 张河牌**（River）→ **再下一轮注**（河牌圈）。
5. **比牌**（Showdown）→ 按牌型分池，牌局结束。

即：**先发 2 张手牌，下注一轮；再发 3 张公共牌，下注一轮；再发转牌，下注一轮；最后发河牌，下注一轮**。实现见 `core/game_logic.py`（`start_new_hand` 发底牌 + 底牌圈，`advance_to_next_stage` 依次发 Flop/Turn/River 并各带一轮下注）。

## 4.1 游戏状态机
游戏核心逻辑应由一个状态机驱动，主要状态包括：
- `WaitingForPlayers`
- `Preflop` (底牌圈)
- `Flop` (翻牌圈)
- `Turn` (转牌圈)
- `River` (河牌圈)
- `Showdown` (比牌)
- `HandOver` (牌局结束)

## 4.2 核心流程
1.  **开始条件**: 桌上玩家 >= 2。
2.  **分配角色**: 确定 Dealer, SB, BB 位置（按座位顺时针）。
3.  **强制下注**: 从 SB, BB 玩家账户扣除盲注。如果玩家筹码不足以支付完整盲注，则该玩家直接 All-in。
4.  **洗牌**: 使用 Fisher-Yates 等公认的随机算法在服务器端洗牌。
5.  **发底牌**: 每人两张。
6.  **下注轮（轮次规则）**:
    -   **底牌圈 (Preflop)**：从 **BB 下家 (UTG)** 开始，顺时针依次行动；**BB 最后行动**（无人加注时 BB 可过牌）。
    -   **翻牌/转牌/河牌圈**：从 **庄位 (BT) 下家** 开始，顺时针依次行动。
    -   顺序始终按 **座位顺序**（player_order / seat index 顺时针）。
    -   服务器验证每个玩家的行动是否合法 (e.g., raise amount is valid)。
    -   一轮下注在「所有未弃牌且未 All-in 的玩家本街投入相同」且「行动回到最后加注者」时结束。
7.  **发出公共牌**: Flop(3), Turn(1), River(1)。
8.  **比牌 (Showdown)**:
    -   服务器为每个未弃牌的玩家，从其2张底牌和5张公共牌中，计算出最大的5张牌组合。
    -   **实现细节**: 必须有一个独立的、经过充分测试的 `HandEvaluator` 模块，用于准确比较任意两手牌的大小。
    -   **Edge Cases**: A-2-3-4-5 是最小顺子；同花大小看最大牌；葫芦大小先看三条。
9.  **底池分配**:
    -   **算法必须支持多个边池 (Side Pots)。**
    -   **流程**:
        1. 遍历所有 All-in 的玩家，按 All-in 金额从小到大排序。
        2. 第一个 (最小的) All-in 玩家定义了第一个主池 (Main Pot)。计算所有玩家投入到这个池的筹码。
        3. 后续的 All-in 玩家定义了 Side Pot 1, Side Pot 2...
        4. 从最后一个 Side Pot 开始，找出有资格赢取该池的玩家中牌最大的，分配奖金。
        5. 逐个向前处理，直到分配完 Main Pot。
        6. 平分底池时，无法均分的奇数筹码，分配给位置最靠前（离Dealer最近）的玩家。
